# راهنمای نصب آفلاین بیواز (سرور بدون اینترنت)

> این راهنما برای سرورهایی است که دسترسی به اینترنت ندارند.
> همه کارها ابتدا روی کامپیوتر شخصی انجام می‌شود، سپس بسته آماده به سرور منتقل می‌شود.

---

## نقشه کلی

```
کامپیوتر شخصی (ویندوز + اینترنت)
    │
    ├─ قدم ۱: نصب Docker Desktop
    ├─ قدم ۲: ساخت بسته آفلاین  ← make-bundle.ps1
    │
    └─ beewaz-bundle.zip  ──────────── USB یا شبکه داخلی ──────────────►
                                                                         │
                                                              سرور (بدون اینترنت)
                                                                  │
                                                                  ├─ قدم ۳: نصب Docker
                                                                  ├─ قدم ۴: انتقال بسته
                                                                  └─ قدم ۵: نصب  ← install-offline.sh
```

---

# بخش اول: کامپیوتر شخصی (ویندوز)

## قدم ۱ — نصب Docker Desktop

اگر Docker Desktop نداری، از سایت رسمی دانلود و نصب کن:
`https://www.docker.com/products/docker-desktop/`

بعد از نصب، Docker Desktop را باز کن و صبر کن تا کاملاً راه‌اندازی شود (آیکون پایین taskbar سبز شود).

---

## قدم ۲ — ساخت بسته آفلاین

پنجره PowerShell را باز کن (دکمه Start → PowerShell):

```powershell
cd C:\Users\me_ahmadi\Music\beewaz-web
powershell -ExecutionPolicy Bypass -File scripts\make-bundle.ps1
```

این اسکریپت:
- اپ را Build می‌کند
- ایمیج‌های PostgreSQL و Nginx را دانلود می‌کند
- همه چیز را در فایل `beewaz-bundle.zip` بسته‌بندی می‌کند

**مدت زمان:** ۵ تا ۱۵ دقیقه بسته به سرعت اینترنت

بعد از اتمام، فایل `beewaz-bundle.zip` در پوشه پروژه ساخته می‌شود.

---

## قدم ۲.۵ — دانلود Docker برای نصب آفلاین روی سرور

اگر سرور هنوز Docker ندارد، باید پکیج‌های Docker را هم از اینترنت دانلود کنی.

### برای Ubuntu 22.04:

```powershell
# یک پوشه بساز
mkdir docker-offline
cd docker-offline
```

از این آدرس‌ها فایل‌های deb را دانلود کن و داخل پوشه `docker-offline` قرار بده:

```
https://download.docker.com/linux/ubuntu/dists/jammy/pool/stable/amd64/
```

فایل‌های مورد نیاز:
- `containerd.io_*.deb`
- `docker-ce_*.deb`
- `docker-ce-cli_*.deb`
- `docker-buildx-plugin_*.deb`
- `docker-compose-plugin_*.deb`

> **راه ساده‌تر:** اگر سرور لینوکس دیگری با اینترنت داری، روی آن Docker را نصب کن و با `apt-get download` پکیج‌ها را دانلود کن:
> ```bash
> apt-get download docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
> ```

این پوشه `docker-offline` را هم در USB کپی کن.

---

## قدم ۳ — انتقال به سرور

### روش USB:
1. فایل `beewaz-bundle.zip` را در USB کپی کن
2. پوشه `docker-offline` را هم (اگر ساختی) در USB کپی کن

### روش شبکه داخلی (SCP):
اگر کامپیوتر شخصی و سرور در یک شبکه هستند:

```powershell
# از PowerShell یا Git Bash
scp beewaz-bundle.zip root@192.168.1.100:/root/
scp -r docker-offline  root@192.168.1.100:/root/
```

## قدم ۴ — نصب Docker روی سرور (اگر نصب نیست)

### بررسی کن Docker هست یا نه:
```bash
docker --version
```

اگر خطا داد، یعنی Docker نصب نیست. ادامه بده:

### نصب از پکیج‌های آفلاین:

```bash
# USB را Mount کن (یا فایل‌ها را از شبکه کپی کرده باشی)
ls /root/docker-offline/

# نصب همه پکیج‌ها با هم
dpkg -i /root/docker-offline/*.deb

# بررسی نصب
docker --version
docker compose version

# فعال کردن سرویس Docker
systemctl enable docker
systemctl start docker
```

## قدم ۵ — آماده‌سازی بسته

```bash
# رفتن به home
cd /root

# اگر از USB منتقل کردی، ابتدا USB را mount کن:
# mount /dev/sdb1 /mnt/usb
# cp /mnt/usb/beewaz-bundle.zip /root/

# استخراج zip
apt install -y unzip 2>/dev/null || yum install -y unzip 2>/dev/null || true
unzip beewaz-bundle.zip -d beewaz

cd beewaz
ls -la
```

باید این فایل‌ها را ببینی:
```
images/
  app.tar
  postgres.tar
  nginx.tar
docker-compose.yml
nginx/
  default.conf
  certs/
.env.example
scripts/
  install-offline.sh
README.txt
```

## قدم ۶ — تنظیم فایل .env

```bash
cp .env.example .env
nano .env
```

این مقادیر را تنظیم کن:

```bash
# رمز دیتابیس — هر چیزی که دوست داری
POSTGRES_PASSWORD=Beewaz@DB#2024

# کلید رمزنگاری — با دستور زیر بساز:
# openssl rand -base64 32
AUTH_SECRET=PASTE_OPENSSL_OUTPUT_HERE

# آدرس سایت — اگر دامنه نداری، IP سرور را بگذار
NEXTAUTH_URL=http://192.168.1.100
NEXT_PUBLIC_APP_URL=http://192.168.1.100

# اطلاعات ادمین
ADMIN_PHONE=09120000000
ADMIN_PASSWORD=Admin@Beewaz#2024
ADMIN_NAME=مدیر سیستم
```

برای ساخت AUTH_SECRET:
```bash
openssl rand -base64 32
# خروجی را کپی کن و در AUTH_SECRET بگذار
```

ذخیره: `Ctrl+X` → `Y` → `Enter`

## قدم ۷ — تنظیم Nginx برای IP (بدون دامنه)

اگر دامنه نداری و فقط با IP دسترسی داری:

```bash
nano nginx/default.conf
```

بخش HTTPS را کامنت کن و این را جایگزین کن:

```nginx
server {
    listen 80;
    server_name _;

    client_max_body_size 20M;

    location /_next/static/ {
        proxy_pass http://app:3000;
        add_header Cache-Control "public, max-age=31536000, immutable";
    }

    location / {
        proxy_pass http://app:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade    $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host       $host;
        proxy_set_header X-Real-IP  $remote_addr;
        proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

> اگر دامنه داری و SSL می‌خواهی، فایل اصلی را نگه دار و SSL certificate در قدم بعد اضافه کن.

---

## قدم ۸ — SSL Certificate (اگر دامنه داری)

اگر certificate داری، فایل‌ها را کپی کن:
```bash
mkdir -p nginx/certs
# certificate های خودت را کپی کن:
cp /path/to/fullchain.pem nginx/certs/
cp /path/to/privkey.pem   nginx/certs/
```

اگر نداری، اسکریپت نصب خودکار self-signed می‌سازد.

## قدم ۹ — نصب

```bash
bash scripts/install-offline.sh
```

این اسکریپت:
1. ایمیج‌های Docker را از فایل tar بارگذاری می‌کند
2. SSL می‌سازد (اگر نداری)
3. همه سرویس‌ها را بالا می‌آورد
4. Migration دیتابیس را اجرا می‌کند

**مدت زمان:** ۲ تا ۵ دقیقه

## آپدیت در آینده (بدون اینترنت)

وقتی نسخه جدیدی داری:

### روی کامپیوتر شخصی:
```powershell
# دریافت آخرین تغییرات از git
cd C:\Users\me_ahmadi\Music\beewaz-web
git pull origin main

# ساخت بسته جدید
powershell -ExecutionPolicy Bypass -File scripts\make-bundle.ps1
```

### روی سرور:
```bash
# جایگزینی بسته جدید
cd /root
unzip -o beewaz-bundle.zip -d beewaz
cd beewaz

# بارگذاری ایمیج جدید
docker load -i images/app.tar

# ری‌استارت
docker compose up -d --no-deps app
```

## دستورات مفید روی سرور

```bash
# وضعیت سرویس‌ها
docker compose ps

# لاگ live
docker compose logs -f app

# ری‌استارت اپ
docker compose restart app

# خاموش کردن کامل
docker compose down

# بکاپ دیتابیس
docker compose exec postgres pg_dump -U beewaz beewaz > backup_$(date +%Y%m%d).sql

# مصرف منابع
docker stats --no-stream
```

## عیب‌یابی

**سایت باز نمی‌شود:**
```bash
docker compose logs nginx
docker compose logs app
# بررسی پورت‌ها:
ss -tlnp | grep -E ':80|:443|:3000'
```

**خطای دیتابیس:**
```bash
docker compose logs postgres
docker compose exec postgres pg_isready -U beewaz
```

**ایمیج پیدا نشد:**
```bash
# بررسی ایمیج‌های موجود
docker images
# بارگذاری دستی
docker load -i images/app.tar
```

**فضای دیسک پر است:**
```bash
df -h
docker system prune -f
```
