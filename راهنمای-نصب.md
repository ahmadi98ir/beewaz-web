# راهنمای قدم به قدم نصب بیواز روی سرور

---

## پیش‌نیازها

- سرور لینوکس (Ubuntu 22.04 یا Debian 12 پیشنهادی)
- حداقل ۱ گیگ RAM و ۱۰ گیگ فضای دیسک
- دامنه‌ای که DNS آن به IP سرور اشاره کند
- دسترسی SSH به سرور

---

## قدم ۱ — اتصال به سرور

از ترمینال خود (یا PuTTY در ویندوز) وارد سرور شو:

```bash
ssh root@YOUR_SERVER_IP
```

## قدم ۳ — نصب Docker

```bash
# نصب پیش‌نیازها
apt install -y ca-certificates curl gnupg

# اضافه کردن کلید Docker
install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor -o /etc/apt/keyrings/docker.gpg
chmod a+r /etc/apt/keyrings/docker.gpg

# اضافه کردن مخزن Docker
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" \
  > /etc/apt/sources.list.d/docker.list

# نصب Docker
apt update
apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# بررسی نصب
docker --version
docker compose version
```

## قدم ۶ — ساختن فایل تنظیمات (.env)

```bash
cp .env.example .env
nano .env
```

مقادیر زیر را **حتماً** تغییر بده:

```
POSTGRES_PASSWORD=یک_رمز_قوی_انتخاب_کن

AUTH_SECRET=اینجا_خروجی_دستور_زیر_را_بگذار

NEXTAUTH_URL=https://beewaz.ir
NEXT_PUBLIC_APP_URL=https://beewaz.ir

ADMIN_PHONE=09120000000
ADMIN_PASSWORD=رمز_ادمین_قوی
ADMIN_NAME=مدیر سیستم
```

برای ساختن AUTH_SECRET یک مقدار تصادفی و امن:

```bash
openssl rand -base64 32
```

خروجی این دستور را در AUTH_SECRET بگذار.

برای ذخیره و خروج از nano: `Ctrl+X` سپس `Y` سپس `Enter`

---

## قدم ۷ — تنظیم نام دامنه در Nginx

فایل nginx را باز کن:

```bash
nano nginx/default.conf
```

خط زیر را پیدا کن و نام دامنه واقعی خود را بگذار:

```
server_name beewaz.ir www.beewaz.ir;
```

ذخیره و خروج: `Ctrl+X` → `Y` → `Enter`

## قدم ۸ — گرفتن گواهی SSL (رایگان با Let's Encrypt)

```bash
# نصب certbot
apt install -y certbot

# گرفتن certificate
# (مطمئن شو پورت 80 باز است و DNS دامنه به این سرور اشاره دارد)
certbot certonly --standalone -d beewaz.ir -d www.beewaz.ir

# کپی certificate ها
mkdir -p nginx/certs
cp /etc/letsencrypt/live/beewaz.ir/fullchain.pem nginx/certs/
cp /etc/letsencrypt/live/beewaz.ir/privkey.pem  nginx/certs/

# تنظیم مجوز خواندن
chmod 644 nginx/certs/*.pem
```

> اگر دامنه هنوز روی این سرور ست نشده، از self-signed certificate استفاده کن:
> ```bash
> mkdir -p nginx/certs
> openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
>   -keyout nginx/certs/privkey.pem \
>   -out nginx/certs/fullchain.pem \
>   -subj '/CN=beewaz.ir'
> ```

## قدم ۹ — اجرای پروژه


## قدم ۱۰ — بررسی سلامت سرویس‌ها

## دستورات مفید

```bash
# خاموش کردن همه سرویس‌ها
docker compose down

# ری‌استارت فقط اپ
docker compose restart app

# ری‌استارت فقط nginx
docker compose restart nginx

# مشاهده مصرف منابع
docker stats

# ورود به محیط دیتابیس
docker compose exec postgres psql -U beewaz -d beewaz

# بکاپ دیتابیس
docker compose exec postgres pg_dump -U beewaz beewaz > backup_$(date +%Y%m%d).sql

# بازگردانی بکاپ
docker compose exec -T postgres psql -U beewaz -d beewaz < backup_20240101.sql
```

## تمدید خودکار SSL

```bash
# تست تمدید
certbot renew --dry-run

# اضافه کردن cron job برای تمدید خودکار
crontab -e
```

خط زیر را اضافه کن (هر روز ساعت ۳ صبح بررسی می‌کند):

```
0 3 * * * certbot renew --quiet && cp /etc/letsencrypt/live/beewaz.ir/fullchain.pem /var/www/beewaz-web/nginx/certs/ && cp /etc/letsencrypt/live/beewaz.ir/privkey.pem /var/www/beewaz-web/nginx/certs/ && docker compose -f /var/www/beewaz-web/docker-compose.yml restart nginx
```

## عیب‌یابی

**سایت باز نمی‌شود:**
```bash
docker compose logs nginx
docker compose logs app
```

**خطای دیتابیس:**
```bash
docker compose logs postgres
docker compose exec postgres pg_isready -U beewaz
```

**build شکست خورد:**
```bash
docker compose build --no-cache app
docker compose up -d
```

**پورت ۸۰ یا ۴۴۳ اشغال است:**
```bash
ss -tlnp | grep -E ':80|:443'
# سرویس مزاحم را متوقف کن، مثلاً:
systemctl stop apache2
```

---

## نکات امنیتی

- رمزهای `.env` را هرگز در git قرار نده
- پورت `5432` (PostgreSQL) را در فایروال ببند
- فقط پورت `80` و `443` باز باشند
- بکاپ منظم از دیتابیس بگیر
- سرور را منظم آپدیت کن: `apt update && apt upgrade -y`

```bash
# تنظیم فایروال ساده
ufw allow 22
ufw allow 80
ufw allow 443
ufw enable
```
